# E2E — Финальная проверка экосистемы

## Контекст

Все фазы V-1..V-3 и H-0..H-7 завершены. Нужно провести полный прогон сценария от регистрации компании в Vitrina до просмотра тепловой карты в Hub. По ходу — выявить и починить мелкие баги.

---

## Сценарий

### Шаг 1: Подготовка тенанта в Vitrina

1. Зайти на `admin.yanbada.com` (или localhost dev)
2. Создать тестовый тенант «Epson Test» (или использовать существующий)
3. `/admin/t/epson-test/profile`:
   - Категории: IT, Производство
   - Теги: принтеры, сканеры, цифровая печать
   - Страна: Япония, Город: Токио
   - Website: epson.com
   - Соцсети: instagram, linkedin
   - Добавить 2 контактных лица
   - Описание: ru + en
4. Сохранить → проверить toast «Сохранено» и появление бейджа «Синхронизировано с Hub»
5. Создать публичную страницу `/p/epson-test` с блоком form и опубликовать

### Шаг 2: Проверка sync в Hub

1. В Supabase Studio открыть `hub.company_cache` → найти строку с tenant_id Epson
2. Проверить что все поля заполнены, `vitrina_page_slug = "epson-test"`
3. Если нет — нажать «Синхронизировать с Hub» вручную на странице профиля

### Шаг 3: Создание события в Hub

1. Зайти на `hub.yanbada.com/organizer/events`
2. Создать событие «Digital Bridge 2025»:
   - Slug: `digitalbridge-2025`
   - Название: ru + en
   - Даты: текущая неделя
   - Город: Алматы
3. Открыть событие → таб «Карта»:
   - Загрузить тестовый SVG-план (можно нарисовать прямоугольник `<svg viewBox="0 0 1000 700"><rect width="1000" height="700" fill="#f0f0f0"/></svg>`)
4. Добавить второй павильон «Hall B» с другим SVG

### Шаг 4: Приглашение участника

1. Таб «Участники» → «Добавить участника»:
   - Email: твой реальный email (или тестовый)
   - Stand number: A-101
   - Pavilion: main
2. Скопировать сгенерированный код доступа
3. Проверить email (или консоль сервера если Resend не настроен)
4. Добавить ещё 3-5 участников через CSV для разнообразия данных

### Шаг 5: Подключение компании к выставке

1. Открыть `/exhibitor/events/join?event=digitalbridge-2025&code={код}`
2. Подтвердить участие
3. Проверить что в `/exhibitor/events` появилась карточка с событием
4. Открыть карточку события → виден стенд A-101

### Шаг 6: Расстановка стендов на карте

1. Вернуться в кабинет организатора → событие → «Открыть редактор карты»
2. Стенд A-101 виден в списке слева как «Не размещён»
3. Перетащить на канвас, разместить
4. Включить snap-to-grid, переместить ещё пару стендов
5. Multi-select (Shift+click) двух стендов → выровнять по левому краю
6. Cmd+D на стенде → создать дубликат
7. Сохранение работает по mouseup

### Шаг 7: Опубликовать событие

1. На странице события — кнопка «Опубликовать»
2. Открыть `/e/digitalbridge-2025/catalog` от лица посетителя (другой браузер, инкогнито)
3. Проверить: карточка Epson присутствует, лого, описание, категории, стенд
4. Поиск по «принтер» → находится Epson (через теги)
5. Фильтр по категории IT → остаётся Epson
6. Клик «Профиль» → открывается `/e/digitalbridge-2025/company/epson-test`
7. Большая кнопка «Открыть полный профиль» → редирект на Vitrina с `?ref=catalog&event=digitalbridge-2025`

### Шаг 8: Карта

1. `/e/digitalbridge-2025/map` — отображается SVG + размещённые стенды
2. Переключение между павильонами через табы
3. Фильтр по категории IT → нерелевантные стенды тускнеют
4. Клик на стенд → Sheet с карточкой компании
5. Поиск «Epson» — стенд пульсирует/подсвечивается

### Шаг 9: QR-код

1. В кабинете организатора → событие → таб Участники → «Печать QR-кодов»
2. Распечатать на бумаге (или сохранить PDF)
3. Отсканировать телефоном → редирект на `vitrina.yanbada.com/p/epson-test?ref=qr&event=digitalbridge-2025`
4. Открывается публичная страница Epson

### Шаг 10: Аналитика

1. Подождать ~5 секунд (события трекинга записываются)
2. Открыть `/organizer/events/digitalbridge-2025/analytics`:
   - Метрики: catalog_view, map_view, profile_view, qr_scan — все > 0
   - График по дням
   - Топ компаний — Epson наверху
   - Pie источников
3. Открыть `/exhibitor/analytics` от лица Epson:
   - Сравнение участий по выставкам
   - Если только одно событие — таблица с одной строкой

### Шаг 11: Тепловая карта

1. В дашборде организатора прокрутить до секции «Тепловая карта»
2. Должны быть точки на местах стендов с активностью
3. Переключить метрику «Только сканирования QR» — точки изменятся
4. Сменить период на 7д — пересчёт
5. Скачать SVG, открыть в браузере — корректно

### Шаг 12: Embed

1. Открыть `/organizer/events/digitalbridge-2025/embed`
2. Скопировать iframe-код
3. Создать тестовую HTML страницу:
   ```html
   <!DOCTYPE html>
   <html><body>
     <h1>Test</h1>
     <iframe src="https://hub.yanbada.com/e/digitalbridge-2025/catalog?embed=1" width="100%" height="800" frameborder="0"></iframe>
   </body></html>
   ```
4. Открыть локально — каталог отображается без шапки Hub
5. Скопировать виджет-код:
   ```html
   <script src="https://hub.yanbada.com/widgets/hub-widget.js" data-event="digitalbridge-2025" async></script>
   <button data-yanbada-hub>Каталог</button>
   ```
6. Клик по кнопке открывает overlay с iframe

### Шаг 13: White-label (опционально, требует DNS)

1. Если есть тестовый домен — прописать `events.settings.custom_domain = test.example.com`
2. Настроить CNAME → cname.vercel-dns.com
3. Добавить домен в Vercel UI
4. Открыть `test.example.com` → редирект на каталог события (middleware делает rewrite)

## Известные проблемы — починить

В процессе E2E могут вылезти:

1. **Race condition при sync**: тенант сохранил профиль до того как Hub запустился — нужно ручной кнопкой синкнуть позже. ОК если работает кнопка.
2. **CSV с не-ASCII**: проверить что русские названия стендов и павильонов сохраняются корректно.
3. **Trailing slash в slug**: проверить `/e/digitalbridge-2025/` vs `/e/digitalbridge-2025` — должно работать одинаково.
4. **Сессия истекла во время drag**: при отправке `PATCH /position` если 401 → показать toast и предложить перелогиниться.
5. **Большие SVG**: SVG карта > 1Mb — должен ли быть лимит? Поставить 2Mb через middleware размер запроса.
6. **Дубликаты track_events**: проверить дедупликацию по session_id для view-событий.
7. **Кэширование `company_cache`**: если Vitrina часто пушит — Hub должен быстро видеть обновления. Проверить что `synced_at` обновляется.
