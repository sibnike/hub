# H-10 — Применение дизайн-системы во все страницы гайда

## Контекст

Гайд посетителя (H-9) работает функционально, но визуально на дефолтном shadcn-стиле. Это плохо.
Документ `docs/DESIGN.md` (приложен) — единственный источник правды по дизайну. Все визуальные решения принимаются по нему.

Цель: переработать визуал всех страниц гайда и публичных страниц события под этот стандарт.

## Что нужно сделать

### 1. Положить `docs/DESIGN.md` в проект

Создать `docs/DESIGN.md` со содержимым приложенного к задаче файла.

### 2. Создать `/components/icons/` со всеми SVG-иконками

Создать минимум 30 иконок согласно DESIGN.md разделу «Иконки → Структура».
Стиль: outline, currentColor, stroke 1.5, viewBox 24x24, размер по умолчанию 20.

Список обязательных:
- Поиск/фильтры: SearchIcon, FilterIcon, CloseIcon, CheckIcon
- Действия: HeartIcon, HeartFilledIcon, StarIcon, StarFilledIcon
- Локации: MapPinIcon, LocationIcon, BuildingIcon
- Навигация: ChevronRightIcon, ChevronLeftIcon, ChevronDownIcon, ChevronUpIcon, ArrowRightIcon, ArrowLeftIcon
- Действия с карточкой: ExternalLinkIcon, HandshakeIcon, EditIcon, TrashIcon, CopyIcon
- Контакты: PhoneIcon, MailIcon, GlobeIcon
- Контент: CategoryIcon, CalendarIcon, ClockIcon, TicketIcon, QrIcon, EyeIcon
- Пользователь: UserIcon, LogoutIcon, SettingsIcon, MenuIcon
- Бонусы и опросы: BonusIcon, PollIcon, CheckCircleIcon
- Карта: ZoomInIcon, ZoomOutIcon
- Будущее: CityGuideIcon (placeholder)

`/components/icons/index.ts` экспортирует все + `iconMap` для динамической замены.

### 3. Подключить шрифты через next/font

Создать `lib/event-fonts.ts` со словарём `fontMap` из 5 пар шрифтов (см. DESIGN.md → «Шрифт-пары»).
В корневом layout гайда читать `event.settings.font_pair` и подключать соответствующие переменные `--font-heading` и `--font-body`.

В `tailwind.config.ts`:
```ts
fontFamily: {
  heading: ['var(--font-heading)', 'var(--font-body)', 'sans-serif'],
  body: ['var(--font-body)', 'sans-serif'],
}
```

### 4. Инжектирование темы события

В `app/e/[slug]/guide/layout.tsx` (а также в публичных страницах события) обернуть контент в div со inline-стилями из `event.settings`:

```tsx
<div
  style={{
    '--accent': event.settings?.accent_color ?? '#3B82F6',
    '--brand': event.settings?.brand_color ?? '#0F172A',
    '--hero-bg': event.settings?.hero_bg ?? 'linear-gradient(135deg, #F8FAFC 0%, #EFF6FF 100%)',
  } as React.CSSProperties}
  className={`${headingFont.variable} ${bodyFont.variable}`}
>
  {children}
</div>
```

### 5. Расширить event.settings новыми полями

Добавить в кабинете организатора `/organizer/events/[slug]` раздел «Брендинг» с полями:
- `accent_color` (color picker)
- `brand_color` (color picker, тёмный для заголовков)
- `hero_bg` — radio: «Градиент» / «Картинка» / «Однотонный»
  - Для картинки: upload поле (Cloudinary)
  - Для градиента: 2 color picker + угол
  - Для однотонного: 1 color picker
- `font_pair` — select из 5 вариантов с превью
- `welcome_message` — i18n textarea (уже добавлено в H-9)
- `brand_logo_url` — upload (уже есть)
- `brand_footer_text` — i18n input (уже есть)

API: `PATCH /api/organizer/events/[slug]/branding` с этими полями.

### 6. Переделать все страницы гайда по DESIGN.md

#### 6.1 Шапка гайда

`components/visitor/guide-header.tsx`:

#### 6.2 Главная гайда `/e/[slug]/guide/page.tsx`

Структура:
1. **Hero-секция** на всю ширину, 60vh desktop / 50vh mobile
   - Фон: `var(--hero-bg)` — может быть градиент или картинка с темной overlay

2. **Tier-блок** — карточка с описанием статуса

3. **Активные опросы** (если есть)
   - Заголовок «Активные опросы» + ссылка «Все →»
   - Горизонтальная карусель (на desktop сетка 2-3) карточек опросов
   - Каждая карточка: вопрос (font-heading), bonus_reward (BonusIcon + N баллов), кнопка «Ответить»

4. **Быстрая навигация** — большие плитки 2×2 или 4 в ряд:
   - Каталог (с иконкой)
   - Карта
   - Избранное (счётчик)
   - Опросы (счётчик активных)

5. **Подборки** (placeholder)
   - Блок «Гид по городу скоро будет доступен»
   - Иконка `CityGuideIcon`, текст приглушённый

## Технические требования

- Tailwind CSS — расширить config переменными `font-heading`, `font-body`, `text-brand`, `text-accent`, `text-muted` и т.д.
- Все цвета только через CSS-переменные
