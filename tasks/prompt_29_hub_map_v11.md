#### Редактор организатора

- В редакторе уже есть селектор павильона (H-3). Расширить:
  - Добавление нового павильона/этажа (кнопка «+ Новый павильон»)
  - Удаление карты (с предупреждением «Стенды на этой карте потеряют позицию»)
  - Изменение названия павильона и номера этажа

API:
- `PATCH /api/organizer/events/[slug]/maps/[mapId]` — обновить pavilion/floor/sort_order

```typescript
export async function PATCH(request, { params }) {
  const { pavilion, floor, sort_order } = await request.json()

  const supabase = await createClient()
  const { data: map } = await supabase.schema('hub').from('event_maps')
    .select('event_id, events:event_id(organizer_tenant_id)')
    .eq('id', params.mapId).single()

  if (!map || !(await assertTenantAdmin(map.events.organizer_tenant_id))) {
    return NextResponse.json({ error: 'Forbidden' }, { status: 403 })
  }

  const { error } = await supabase.schema('hub').from('event_maps').update({
    pavilion, floor, sort_order,
  }).eq('id', params.mapId)

  if (error) return NextResponse.json({ error: error.message }, { status: 500 })
  return NextResponse.json({ ok: true })
}
```

При смене павильона/этажа карты — стенды этой карты НЕ переезжают автоматически. Они остаются со старыми `pavilion`/`floor`. Это нужно подсветить в UI.

### 4. Улучшения редактора

#### Snap-to-grid

Опциональная сетка с шагом 1% (или 0.5%) — переключатель в панели редактора.
При drag — координаты округляются:

```typescript
function snap(value: number, gridSize: number = 1): number {
  return Math.round(value / gridSize) * gridSize
}

const newX = snapEnabled ? snap(rawX, gridSize) : rawX
```

Если snap включён — рисовать визуальную сетку поверх SVG (lines через `<svg>` overlay) с opacity 0.1.

#### Выравнивание выделенных стендов

Multi-select стендов: Shift+click добавляет стенд в выделение.
Кнопки в панели: выровнять по левому/правому краю, по верху/низу, распределить равномерно.

```typescript
// Выровнять по левому краю — найти минимальный map_x, остальным присвоить его
function alignLeft(selected: Stand[]) {
  const minX = Math.min(...selected.map(s => s.map_x))
  return selected.map(s => ({ ...s, map_x: minX }))
}
```

После выравнивания — batch update API:
- `POST /api/organizer/events/[slug]/stands/batch-position`
- body: `{ updates: [{ standId, map_x, map_y, map_width, map_height }, ...] }`

#### Копирование стендов

Cmd/Ctrl+D на выделенном стенде — дубликат со смещением 2% вправо/вниз.
Полезно когда у компании несколько одинаковых стендов рядом.

> Внимание: дубликат — это **новый event_stand для той же participation**.
> Один participation может иметь несколько stands (например двойной стенд).
> Проверь что схема это позволяет — `event_stands.participation_id` не уникален.

#### Превью масштаба

В углу карты — мини-индикатор «1 см ≈ X м» (необязательно, если есть метаданные масштаба в SVG).

### 5. Управление картой как файлом

#### Замена SVG карты

Сейчас при загрузке новой SVG создаётся новая запись `event_maps`. Добавить возможность:
- На существующей карте — кнопка «Заменить SVG» → текстарея с текущим содержимым → сохранение через PATCH

```typescript
// PATCH /api/organizer/events/[slug]/maps/[mapId]
// body: { svg_content: string }
```

При замене SVG — стенды НЕ перемещаются (их координаты в %, поэтому относительно нового SVG они окажутся в тех же относительных позициях). Это правильное поведение.

#### Экспорт карты с расставленными стендами

Кнопка «Экспортировать как SVG» в редакторе:
- Берёт `svg_content` карты
- Перед `</svg>` вставляет `<rect>` для каждого размещённого стенда с координатами и номером
- Скачивается как файл `{event-slug}-{pavilion}-{floor}.svg`

Полезно для печати или передачи дизайнеру.
