# ROADMAP — Exhibitor Hub, следующие фазы

Статус: на проде работают все фазы H-0...H-8 включительно.
Базовый поток выставки работает end-to-end: организатор создаёт событие, добавляет участников,
расставляет стенды на карте, генерирует QR. Посетитель находит компанию в каталоге или на карте,
сканирует QR, попадает на страницу Vitrina. Аналитика собирается, тепловая карта рисуется,
embed/white-label работают.

## Инварианты (сохранять)

- Hub никогда не хранит данные компании — только `tenant_id` и кэш через webhook.
- Hub никогда не пишет в схему `public` (только читает по tenant_id).
- Все cross-schema запросы — через ручной JOIN (`joinTenants()`).
- Все RLS-политики через SECURITY DEFINER функции.
- SVG из пользовательского ввода — санитизировать через `sanitize-html` (не `isomorphic-dompurify`).
- Трекинг (`/api/track`) — fire-and-forget, не блокировать рендер.
- Маршруты карты/каталога — `force-dynamic`, без кеша.

## Фаза H-10 — Кабинет гостя посетителя

Сейчас посетитель — анонимный (избранное только в localStorage). Добавить полноценный кабинет.

### Авторизация

- Вход по magic-link на email на странице `/g/login` (отдельно от tenant-кабинета)
- Отдельный контур доступа, аналогично `/s/*` у staff в Vitrina
- При первом входе — короткая форма: имя, страна, должность (опционально)
- Возможность подключаться к нескольким событиям одновременно

### Структура

```
/g/login                       вход по magic-link
/g/                            обзор всех мероприятий
/g/e/[event-slug]/             моё участие в конкретном событии
    favorites/                 избранные компании
    meetings/                  запрошенные встречи
    notes/                     заметки о компаниях
```

### Модель данных

- `hub.guests` (id, email, name, country, role, created_at)
- `hub.guest_event_attendance` (guest_id, event_id, joined_at) — связь many-to-many
- `hub.guest_favorites` (guest_id, event_id, tenant_id, saved_at) — мигрировать из localStorage при логине
- `hub.guest_notes` (guest_id, tenant_id, event_id, note text, met_status: planned|met|skipped, met_at)

### UI

- Главная: «Мои выставки» — карточки событий куда я добавлен
- В событии: «Мои избранные» с возможностью пометить «уже встретился» / «планирую встретиться»
- Заметки о компании прямо из карточки
- Экспорт списка контактов после выставки в CSV

## Фаза H-11 — Запросы и назначение встреч

Сейчас кнопка «Встреча» — заглушка. Раскрыть её.

### Поток

```
Гость в каталоге/на карте → кнопка «Запросить встречу»
    → форма: предлагаемое время, тема, сообщение
    → создаётся meeting_request (status: pending)
    → email компании-участнику + уведомление в кабинете Hub
    → компания одобряет/предлагает другое время/отклоняет
    → подтверждённая встреча видна обоим в календаре кабинета
```

### Модель

- `hub.meeting_requests`:
  - id, event_id, guest_id (или contact_email если без аккаунта)
  - tenant_id (с кем хотят встретиться)
  - proposed_time_from, proposed_time_to
  - topic, message
  - status: pending | accepted | rescheduled | rejected | completed
  - response_message, response_time_from, response_time_to
  - created_at, responded_at

### UI

- В кабинете компании-участника — раздел «Входящие запросы на встречи»
- В кабинете гостя — раздел «Мои встречи» (запрошенные, подтверждённые, прошедшие)
- На странице события у организатора — общий счётчик встреч (метрика популярности)
- При подтверждённой встрече — кнопка «Добавить в Google Calendar» (ics-файл)

### Email-уведомления

- Запрос → email компании
- Подтверждение/изменение → email гостю
- За 1 час до встречи → email обоим
- После встречи → email гостю с просьбой поставить отметку «состоялась»

### Аналитика

- Для компании-участника: количество запросов, конверсия в подтверждённые, статус «состоялась»
- Для гостя: его статистика встреч (запрошено, подтверждено, состоялось)
- Для организатора: топ компаний по запросам встреч (метрика популярности)

## Фаза H-15 — После события (post-event)

Когда событие закончилось — что дальше?

- Архивный режим события (read-only, исторические данные)
- Гость получает финальный email с экспортом контактов которых добавил в избранное
- Компания-участник видит итоговую аналитику (сравнение с другими своими выставками)
- Опросы NPS — оценить событие и компании
- «Возможно вам будет интересно» — другие предстоящие события

## Технический долг

- Magic-link при регистрации в Vitrina должен сразу логинить — иногда требуется второй вход
- Дедупликация в `track_events` сейчас по session_id, но при agressive Open Graph краулерах может ложно срабатывать — нужна проверка User-Agent
- Авто-высота iframe в карточке компании иногда долго догоняет (visual jump при первом рендере) — можно показывать скелетон
- Тепловая карта при большом количестве событий (10к+) тяжёлая — рассмотреть агрегацию на бэке в `event_analytics` через ежедневный cron
- Embed-виджет (`hub-widget.js`) сейчас минифицирован вручную — настроить build pipeline
